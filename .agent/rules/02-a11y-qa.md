# Accessibility + UX QA Rules

Obrigatório:
1) Navegação por teclado completa:
   - Tab/Shift+Tab percorre itens em ordem lógica.
   - focus-visible sempre aparente.
2) Inputs precisam de:
   - label associado
   - aria-* quando necessário
   - mensagens de erro legíveis e com papel semântico.
3) Links devem ser distinguíveis (não depender só de cor).
4) Tamanho de alvo (click/tap): preferir >= 40px em botões/itens de navegação.
5) Em telas com dados “0”/vazio:
   - criar empty state com orientação clara (não só “0”).
6) Produzir checklist final de QA com:
   - contraste, foco, estados, responsividade, consistência, microcopy em pt-BR.
